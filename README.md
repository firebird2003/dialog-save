# dialog-save

将你与 AI 代理的对话自动保存为 Markdown 文件，支持 Obsidian 笔记库查看，多主机通过 WebDAV 同步。

## 功能特性

- 📝 自动或手动保存对话为 Markdown 格式
- 🗂️ 智能话题识别，自动分割不同主题的对话
- 📁 兼容 iCloud/Obsidian 同步的目录结构
- 🔍 frontmatter 元数据支持，方便检索
- 🌐 WebDAV 服务，多主机同步
- 💾 离线缓存，网络恢复后自动同步

## 快速开始

```bash
cd ~/.openclaw/workspace/skills && git clone https://github.com/firebird2003/dialog-save.git && cd dialog-save && bash scripts/manage.sh
```

## 目录结构

```
Obsidian笔记库/
├── 管理者@yejimaca2141/    # 代理名@主机名（兼容iCloud）
│   └── 260316/
│       └── 2603162100+对话主题.md
└── 其他笔记.md
```

**目录名格式**：`代理名@主机名`（使用 `@` 符号）

## 保存机制

### 自动模式
- 自动保存所有对话
- 无需用户提示
- 适合完整记录

### 手动模式
- 需要说"存入本地目录"或"保存对话"
- 智能话题识别
- 适合选择性保存

## 使用方法

**手动模式**：对话中说以下短语
- "存入本地目录"
- "保存对话"
- "存档"

**自动模式**：无需任何操作

## 配置项

| 配置 | 说明 | 选项 |

| Obsidian 路径 | 笔记库根目录 | 路径 |

| 代理名称 | 对话来源标识 | 自定义 |
| 主机名称 | 对话来源标识 | 自动检测 |
| 保存机制 | 自动/手动 | auto/manual |

| 开机自启 | 服务自启动 | true/false |

## iCloud 兼容说明

### 问题原因

iCloud Drive 同步时会检测目录名冲突，使用不同分隔符（如 `-`）创建的目录可能与云端版本冲突，导致目录消失。

### 解决方案

- 目录名使用 `@` 符号：`代理名@主机名`
- 与现有 iCloud 同步结构保持一致
- 避免中间子目录

## 技术实现

| 组件 | 技术 |
|------|------|
| WebDAV | rclone |
| 配置 | JSON |
| 脚本 | Bash |
| 格式 | Markdown + YAML frontmatter |
